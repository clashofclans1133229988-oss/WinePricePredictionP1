📋 STEP 2: RUN THE FLASK APP
================================================================================

STARTING THE SERVER:
    
    Command: python app.py
    
    What happens:
    ═════════════
    1. Python reads your app.py file
    2. Flask creates a web server
    3. Server starts listening on localhost:5000
    4. Terminal shows:
    
       ======================================================================
       🚀 FLASK SERVER STARTING...
       ======================================================================
       📍 Server running at: http://localhost:5000
       🏠 Visit home page at: http://localhost:5000/
       🔮 Send predictions to: http://localhost:5000/predict_api
       ⚠️  Use POST method in Postman to send JSON data
       ======================================================================

IMPORTANT:
    ⚠️  KEEP THIS TERMINAL OPEN!
    
    If you close this terminal, the Flask server STOPS
    Postman and your browser won't be able to connect
    
    Leave it running while you test
    To stop the server, press: Ctrl + C (in the terminal)

ERROR TROUBLESHOOTING:
    
    Error: "Address already in use"
    ├─ Cause: Port 5000 already in use by another app
    ├─ Fix: Close other apps or change port in app.py
    
    Error: "No such file or directory: WINE_PRICE.csv"
    ├─ Cause: Wrong folder path
    ├─ Fix: Make sure you navigated to correct folder before running
    
    Error: "ModuleNotFoundError: No module named 'flask'"
    ├─ Cause: Packages not installed
    ├─ Fix: Run: pip install -r requirements.txt (from Step 1)
    
    Error: "No such file: randomforest_model.pkl"
    ├─ Cause: Flask can't find the pickle files
    ├─ Fix: Verify folder structure is correct (see Step 1)

📋 STEP 6: HOW TO USE POSTMAN (ADVANCED)
================================================================================

WHAT IS POSTMAN?
    Postman is a tool to test APIs without building a webpage
    Think of it as a "messenger" that sends data to your server

INSTALL POSTMAN:
    
    Step 1: Visit: https://www.postman.com/downloads/
    Step 2: Click "Download" for your operating system
    Step 3: Install it (like any normal Windows app)
    Step 4: Open Postman

CREATE A NEW REQUEST:

    Step 1: Open Postman
    
    Step 2: Click "+ New" button (top left)
            OR click "Create" on the left sidebar
    
    Step 3: In the dropdown that appears, click "HTTP Request"
    
    Step 4: You'll see a blank request form

CONFIGURE THE REQUEST:

    Step 1: Look at the dropdown that says "GET"
            Change it to "POST"
            (POST = sending data TO server)
    
    Step 2: In the URL field (next to POST), paste:
            http://localhost:5000/predict_api
    
    Step 3: Click the "Body" tab (below URL)
    
    Step 4: Select "raw" radio button on the left
    
    Step 5: On the right side, change "Text" dropdown to "JSON"
    
    Step 6: In the large text area, paste this sample JSON data:

    ────────────────────────────────────────────────────────

    {
      "Alcohol_log": 2.5,
      "Volatile_Acidity_log_boxcox": 0.8,
      "Citric_Acid": 0.3,
      "Chlorides": 0.05,
      "pH": 3.2,
      "Sulfites": 15,
      "Alcohol_to_Acidity": 5.0,
      "Volatile_to_Fixed_Acidity": 0.4,
      "Alcohol_x_CitricAcid": 0.75,
      "ResidualSugar_x_Alcohol": 2.0,
      "pH_squared": 10.24
    }

    ────────────────────────────────────────────────────────

SEND THE REQUEST:

    Step 1: Click the blue "Send" button on the right
    
    Step 2: Wait 1-2 seconds
    
    Step 3: Below your request, you'll see the response:

    ────────────────────────────────────────────────────────

    {
        "success": true,
        "message": "✅ Prediction successful!",
        "input_features": {
            "Alcohol_log": 2.5,
            "Volatile_Acidity_log_boxcox": 0.8,
            ...
        },
        "predicted_price": 45.73,
        "price_formatted": "$45.73"
    }

    ────────────────────────────────────────────────────────

    Status: 200 OK (green) = SUCCESS!

TEST WITH DIFFERENT VALUES:

    Change any value in the JSON and click "Send" again
    Example: Change "Alcohol_log" from 2.5 to 3.5
    
    The predicted price should change!
    
    Try different values to see how the model responds

📋 COMMON PROBLEMS & SOLUTIONS
================================================================================

PROBLEM 1: Browser shows "Connection refused"
─────────────────────────────────────────────
Error Message: 
    "This site can't be reached"
    "localhost refused to connect"

Cause:
    Flask server is not running

Solution:
    1. Go back to PowerShell terminal
    2. Make sure you ran: python app.py
    3. Check if it shows "Server running at: http://localhost:5000"
    4. If not, run the command again


PROBLEM 2: "ModuleNotFoundError: No module named 'flask'"
───────────────────────────────────────────────────────────
Error Message:
    ModuleNotFoundError: No module named 'flask'

Cause:
    Flask package not installed

Solution:
    1. Stop Flask (Ctrl + C in terminal)
    2. Run: pip install -r requirements.txt
    3. Wait for "Successfully installed..."
    4. Run: python app.py again


PROBLEM 3: Postman shows "Could not get any response"
──────────────────────────────────────────────────────
Cause:
    Flask server stopped or URL is wrong

Solution:
    1. Check Flask is still running in terminal
    2. Check URL is exactly: http://localhost:5000/predict_api
    3. Make sure Body is set to: raw, JSON
    4. Make sure Method is: POST (not GET)


PROBLEM 4: Browser shows 404 Not Found
─────────────────────────────────────
Cause:
    Wrong URL

Solution:
    Use exact URL: http://localhost:5000/
    Note: / at the end is important!


PROBLEM 5: Form doesn't respond when clicking "Predict Price"
──────────────────────────────────────────────────────────────
Cause:
    Missing required fields or typo in feature names

Solution:
    1. Fill ALL 11 fields
    2. Use numeric values only
    3. No empty fields allowed


PROBLEM 6: Error in Flask terminal: "Address already in use"
────────────────────────────────────────────────────────────
Cause:
    Another app is using port 5000

Solution:
    Option A: Close other apps
    Option B: Kill the process using port 5000
             Command: Get-Process -Name python | Stop-Process
    Option C: Change port in app.py (line: app.run(..., port=5000))
             Change 5000 to 5001 or 8000

📋 WHAT EACH FILE DOES
================================================================================

app.py
    ├─ Main Flask application
    ├─ Loads the trained model (randomforest_model.pkl)
    ├─ Handles HTTP requests (GET /, POST /predict_api)
    ├─ Processes input data
    ├─ Makes predictions
    └─ Returns results as JSON

templates/home.html
    ├─ The web page users see
    ├─ Contains the input form
    ├─ Beautiful Lebanese-themed design
    ├─ Has JavaScript to handle form submission
    └─ Displays prediction results

requirements.txt
    ├─ List of Python packages needed
    ├─ When you run: pip install -r requirements.txt
    └─ It installs everything automatically

WinePricePredictionP1/randomforest_model.pkl
    ├─ The trained Machine Learning model
    ├─ Created by the Jupyter notebook earlier
    ├─ Flask loads this to make predictions
    └─ Binary file (compiled Python object)

WinePricePredictionP1/rf_features.pkl
    ├─ List of feature names in correct order
    ├─ Ensures features are sent to model in right order
    └─ Critical for accurate predictions

code.py
    ├─ Utility script to view pickle file contents
    ├─ Not needed for running the app
    └─ Good for debugging

FLASK_API_GUIDE.txt
    ├─ Comprehensive documentation about Flask and APIs
    ├─ Reference material if you want to learn more
    └─ Not needed to run the app

QUICK_START_GUIDE.txt
    └─ This file! Step-by-step instructions
