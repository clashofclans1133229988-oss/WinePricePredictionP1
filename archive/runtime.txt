# WHAT IS THIS FILE?
# ─────────────────
# This file tells Heroku which Python version to use when running your app
# Without this file, Heroku might use a different Python version than expected
# 
# WHY IS IT IMPORTANT?
# ────────────────────
# Different Python versions have different:
#   • Syntax support
#   • Built-in libraries
#   • Performance
#   • Package compatibility
# 
# If Heroku uses Python 3.11 but your code was built for Python 3.7,
# you might get errors like:
#   • ImportError: module not found
#   • SyntaxError: invalid syntax
#   • Version mismatch errors
# 
# By specifying runtime, you GUARANTEE consistency:
#   • Your machine: Python 3.7.16 ✓
#   • Heroku server: Same Python 3.7.16 ✓
#   • No version conflicts! ✓

python-3.7.16

# =====================================================================
# HOW HEROKU USES THIS FILE
# =====================================================================
# 
# Deployment Process:
# 
# 1. You push your code to Heroku
#    └─ Command: git push heroku main
# 
# 2. Heroku reads runtime.txt
#    └─ Sees: python-3.7.16
# 
# 3. Heroku sets up Python 3.7.16 on server
#    ├─ Downloads Python 3.7.16 binary
#    ├─ Installs it on the server
#    └─ Configures it as default
# 
# 4. Heroku reads requirements.txt
#    ├─ Uses Python 3.7.16 to install packages
#    ├─ pip (from Python 3.7.16) installs each package
#    └─ All packages compatible with Python 3.7.16
# 
# 5. Heroku runs your Procfile
#    └─ Command: gunicorn app:app (using Python 3.7.16)
# 
# =====================================================================
# WHAT IF YOU OMIT runtime.txt?
# =====================================================================
# 
# If you don't include runtime.txt:
#   ├─ Heroku picks a DEFAULT Python version
#   ├─ Usually the LATEST version available
#   ├─ Example: Might be Python 3.12 instead of 3.7
#   └─ Results in INCOMPATIBILITY! ❌
# 
# Problems that occur:
#   • Your code uses old syntax not supported in Python 3.12
#   • Packages installed for Python 3.7 fail in Python 3.12
#   • Model pickle files might have format differences
#   • App crashes on Heroku!
# 
# Solution: Always include runtime.txt! ✅
# 
# =====================================================================
# REAL-WORLD EXAMPLE
# =====================================================================
# 
# WITHOUT runtime.txt (WRONG):
