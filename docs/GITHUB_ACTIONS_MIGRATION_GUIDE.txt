📚 PART 2: STEP-BY-STEP - DELETE HEROKU WORKFLOW
================================================================================

STEP 1: IDENTIFY THE FILE
────────────────────────

Your current file (to delete):
  .github/workflows/deploy.yml

This is the Heroku deployment workflow.


STEP 2: DELETE THE FILE
────────────────────────

Option A: Delete in VS Code
  1. In VS Code, open .github/workflows/deploy.yml
  2. Right-click → Delete
  3. Confirm deletion
  4. File is gone!

Option B: Delete via command line
  $ cd path/to/project
  $ rm .github/workflows/deploy.yml

Option C: Delete on GitHub
  1. Go to repository on GitHub
  2. Click .github/workflows/deploy.yml
  3. Click delete button (trash icon)
  4. Confirm deletion


STEP 3: COMMIT AND PUSH
──────────────────────

In terminal:

  $ cd path/to/project
  $ git add .
  $ git commit -m "Remove Heroku deployment workflow"
  $ git push origin main

Or in VS Code:
  1. Click Source Control (left sidebar)
  2. See deleted file marked with "D"
  3. Stage all changes
  4. Commit with message: "Remove Heroku deployment workflow"
  5. Push to GitHub


STEP 4: VERIFY ON GITHUB
─────────────────────────

On GitHub.com:
  1. Go to your repository
  2. Click .github/workflows/
  3. Should see:
     ✅ ci.yml (testing)
     ✅ docker.yml (Docker)
     ❌ deploy.yml (GONE!)


================================================================================
📚 PART 3: ADD NEW RAILWAY WORKFLOW (OPTIONAL)
================================================================================

Why optional?
  ├─ Railway watches Docker Hub automatically
  ├─ New Docker pushes trigger Railway redeploy
  ├─ Workflow just adds notification/verification
  └─ Nice to have, not required

If you deleted deploy.yml, you have two workflows left:
  ├─ CI Testing workflow (tests code)
  ├─ Docker Build workflow (builds & pushes image)

Railway will auto-redeploy when Docker image is pushed!

But if you want verification and notification:

STEP 1: ADD NEW WORKFLOW FILE
──────────────────────────────

Create file: .github/workflows/deploy-railway.yml

Content: (I already created this for you!)
  [See the deploy-railway.yml file I created earlier]


STEP 2: ADD GITHUB SECRET
──────────────────────────

Go to GitHub:
  1. Settings → Secrets and variables → Actions
  2. Click "New repository secret"
  3. Name: RAILWAY_PROJECT_NAME
  4. Value: your-railway-project-name
     └─ This is the name you gave in Railway
     └─ e.g., "wine-price-app"
  5. Click "Add secret"


STEP 3: COMMIT AND PUSH
──────────────────────

$ git add .github/workflows/deploy-railway.yml
$ git commit -m "Add Railway deployment workflow"
$ git push origin main


STEP 4: VERIFY
───────────────

On GitHub:
  1. Click "Actions" tab
  2. Should see three workflows:
     ✅ CI Testing
     ✅ Build and Push Docker Image
     ✅ Trigger Railway Deployment


STEP 5: TEST DEPLOYMENT
────────────────────────

Make a small test change:
  $ git checkout -b test/railway-deploy
  $ echo "# test" >> README.md
  $ git add README.md
  $ git commit -m "Test Railway deployment"
  $ git push origin test/railway-deploy

On GitHub:
  1. Create pull request
  2. CI Testing runs ✅
  3. Merge pull request
  4. Docker Build runs ✅
  5. Railway Deploy workflow runs ✅
  6. Check Railway dashboard
  7. See app deployed!

📚 PART 4: ALTERNATIVE - ADD FLY.IO WORKFLOW
================================================================================

If prefer Fly.io over Railway:

STEP 1: CREATE FLY.IO ACCOUNT & APP
────────────────────────────────────

$ fly launch
  └─ Creates Fly account and app

OR on https://fly.io manually


STEP 2: CREATE FLY API TOKEN
──────────────────────────────

$ fly tokens create deploy
  └─ Copy the token


STEP 3: ADD GITHUB SECRETS
────────────────────────────

Go to GitHub Settings → Secrets:
  1. Secret 1:
     Name: FLY_API_TOKEN
     Value: [token you created]
  
  2. Secret 2:
     Name: FLY_APP_NAME
     Value: your-fly-app-name


STEP 4: ADD FLY WORKFLOW
─────────────────────────

Create file: .github/workflows/deploy-fly.yml

Content: (I already created this for you!)
  [See the deploy-fly.yml file I created earlier]


STEP 5: DELETE HEROKU & RAILWAY WORKFLOWS (if using Fly only)
──────────────────────────────────────────────────────────────

If using Fly instead of Railway:
  $ rm .github/workflows/deploy.yml
  $ rm .github/workflows/deploy-railway.yml

Keep only:
  ├─ ci.yml
  ├─ docker.yml
  └─ deploy-fly.yml


STEP 6: COMMIT AND PUSH
──────────────────────

$ git add .
$ git commit -m "Setup Fly.io automatic deployment"
$ git push origin main


STEP 7: TEST
─────────────

Make test change and push:
  1. GitHub tests code (ci.yml)
  2. Builds Docker (docker.yml)
  3. Deploys to Fly.io (deploy-fly.yml)
  4. Check Fly.io dashboard
  5. App live at: https://your-app-name.fly.dev

📚 PART 5: COMPARISON - WHICH DEPLOYMENT WORKFLOW?
================================================================================

                                  Delete: deploy.yml, deploy-railway.yml

                                Add: deploy-railway.yml
                                Add: deploy-fly.yml
                                Delete: deploy.yml

Already invested in Railway     → Just remove Heroku
No changes needed               Keep: ci.yml, docker.yml
Docker pushing to Hub           Delete: deploy.yml
Railway auto-updates            (Railway watches Docker Hub)

Already invested in Fly.io      → Add Fly deployment
Want automatic CI/CD            Keep: ci.yml, docker.yml
Full automation                 Add: deploy-fly.yml
                                Delete: deploy.yml
