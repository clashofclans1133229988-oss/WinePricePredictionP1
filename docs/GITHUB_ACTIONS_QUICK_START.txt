🚀 THE 5-MINUTE SETUP
================================================================================

STEP 1: YOUR PROJECT IS ALREADY SET UP! ✅ (30 seconds)
─────────────────────────────────────────

You should already have these files:
  ├─ .github/workflows/ci.yml ✅
  ├─ .github/workflows/docker.yml ✅
  ├─ .github/workflows/deploy.yml ✅
  ├─ Dockerfile ✅
  ├─ docker-compose.yml ✅
  ├─ requirements.txt ✅
  └─ app.py ✅

If any missing, get them from your project directory!

STEP 2: PUSH TO GITHUB (1 minute)
──────────────────────────────

In terminal:
  $ cd path/to/project
  $ git add .
  $ git commit -m "Add GitHub Actions workflows"
  $ git push origin main

Verify on GitHub.com that files are there!

STEP 3: ADD 5 SECRETS TO GITHUB (2 minutes)
──────────────────────────────────────────

On GitHub:
  1. Go to Settings → Secrets and variables → Actions
  2. Click "New repository secret"
  3. Add EACH of these:

Secret 1: DOCKER_USERNAME
  Value: your-docker-hub-username

Secret 2: DOCKER_PASSWORD
  Value: [Docker Hub access token]
  Get from: https://hub.docker.com/settings/security

Secret 3: HEROKU_API_KEY
  Value: [Your Heroku API key]
  Get from: https://dashboard.heroku.com → Account settings

Secret 4: HEROKU_EMAIL
  Value: your-email@example.com

Secret 5: HEROKU_APP_NAME
  Value: your-heroku-app-name
  Get/create from: https://dashboard.heroku.com

STEP 4: TEST IT! (1 minute)
───────────────────────────

Make a tiny change:
  $ git checkout -b test/workflow
  $ echo "# test" >> README.md
  $ git add README.md
  $ git commit -m "Test workflow"
  $ git push origin test/workflow

On GitHub:
  1. Make a pull request
  2. Watch "CI Testing" run
  3. Click it to see logs
  4. Should turn ✅ green in 1-2 minutes

STEP 5: MERGE & WATCH DEPLOYMENT (1 minute)
────────────────────────────────────────────

Once CI passes:
  1. Click "Merge pull request"
  2. Delete branch
  3. Watch "Build and Push Docker" workflow
  4. Then watch "Deploy to Heroku" workflow
  5. App live at: https://your-app-name.herokuapp.com

DONE! 🎉

================================================================================
🆘 QUICK FIXES IF SOMETHING FAILS
================================================================================

If CI Testing fails:
  └─ Check Python code syntax
  └─ Run locally: python -m py_compile app.py

If Docker build fails:
  └─ Test locally: docker build -t test .
  └─ Check Docker Hub credentials

If Heroku deploy fails:
  └─ Check Heroku app exists
  └─ Check HEROKU_API_KEY is valid
  └─ View Heroku logs in dashboard

See GITHUB_ACTIONS_TROUBLESHOOTING.txt for detailed help!

================================================================================
❓ COMMON QUESTIONS
================================================================================

Q: How often do workflows run?
A: Every time you push code or create a pull request

Q: Can I disable a workflow?
A: Yes, Settings → Actions → disable specific workflow

Q: Where are my secrets stored?
A: Encrypted in GitHub, never shown in logs

Q: Can I manually trigger a workflow?
A: Yes, Actions tab → click workflow → "Run workflow"

Q: How much does this cost?
A: Free tier: 2000 minutes/month (usually plenty)

Q: Can I see workflow history?
A: Yes, Actions tab shows all past runs

Q: What if I want to deploy without tests passing?
A: Make the action skippable with "continue-on-error: true"

Q: Can I schedule workflows?
A: Yes, use cron syntax in workflow "on" section
