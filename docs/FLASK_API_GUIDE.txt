📚 PART 1: WHAT IS FLASK?
================================================================================

Flask is a Python web framework - basically a toolkit for building web servers.

ANALOGY:
--------
Think of Flask like a restaurant:
  - Customer (Browser/Postman) = Person ordering food
  - Flask app = Restaurant that takes orders
  - Routes (@app.route) = Different service counters
  - Functions = Chefs that prepare the food
  - Response = Meal served back to customer

WHAT CAN FLASK DO?
------------------
1. Render HTML pages (serve web pages to browsers)
2. Create APIs (accept data, process it, return results)
3. Handle different types of requests (GET, POST, PUT, DELETE)
4. Return responses in different formats (HTML, JSON, XML)

FLASK VS EXPRESS.JS:
-------------------
Flask (Python) ≈ Express.js (Node.js)
Both serve the same purpose, just different languages

📚 PART 2: WHAT IS AN API?
================================================================================

API stands for: Application Programming Interface

SIMPLE EXPLANATION:
-------------------
An API is a way for different programs to talk to each other.

ANALOGY:
--------
Restaurant API:
  - Input: Customer sends order (menu item + quantity)
  - Process: Chef prepares the food
  - Output: Server returns the meal

Wine Price API:
  - Input: Send wine features as JSON
  - Process: ML model predicts price
  - Output: Return predicted price as JSON

WHY USE APIs?
-----------
1. Separate frontend (what users see) from backend (data processing)
2. Allow different apps to communicate
3. Enable mobile apps to get data from web servers
4. Enable Postman to test the service

📚 PART 3: REQUEST TYPES (HTTP METHODS)
================================================================================

There are 4 main HTTP request types:

1. GET
   - Purpose: RETRIEVE data from server
   - Example: Get all users from database
   - URL: http://site.com/get_users
   - Data: Sent in URL as parameters
   - Usage in code: @app.route('/route', methods=['GET'])
   - When to use: Getting information, retrieving files, displaying pages

2. POST
   - Purpose: SEND data TO server (create new data)
   - Example: Send a wine prediction request
   - URL: http://site.com/predict_api
   - Data: Sent in request BODY (hidden, not in URL)
   - Usage in code: @app.route('/route', methods=['POST'])
   - When to use: Submitting forms, creating predictions, sending sensitive data
   - ✅ THIS IS WHAT WE USE FOR PREDICTIONS

3. PUT
   - Purpose: UPDATE existing data
   - Example: Update a user's profile
   - URL: http://site.com/user/5
   - Data: Sent in request BODY
   - When to use: Modifying existing records

4. DELETE
   - Purpose: DELETE data
   - Example: Delete a user account
   - URL: http://site.com/user/5
   - When to use: Removing records

FOR OUR WINE PRICE API, WE USE POST BECAUSE:
  ✅ We're sending data to the server
  ✅ Data is secure (not visible in URL)
  ✅ We can send large amounts of data
  ✅ It's professional and standard practice

📚 PART 4: JSON FORMAT (How Data is Sent)
================================================================================

JSON = JavaScript Object Notation
It's the standard format for sending data over the internet.

JSON STRUCTURE:
----------------
{
  "key": "value",
  "number_key": 123,
  "boolean_key": true,
  "array_key": [1, 2, 3]
}

RULES:
------
✓ Keys must be in double quotes: "Alcohol_log"
✓ String values must be in double quotes: "hello"
✓ Numbers don't need quotes: 123 or 2.5
✓ Boolean values: true or false (lowercase, no quotes)
✓ Use commas between key-value pairs
✓ Last item has NO comma (common mistake!)

EXAMPLE FOR OUR WINE API:
--------------------------
{
  "Alcohol_log": 2.5,
  "Volatile_Acidity_log_boxcox": 0.8,
  "Citric_Acid": 0.3,
  "Chlorides": 0.05,
  "pH": 3.2,
  "Sulfites": 15,
  "Alcohol_to_Acidity": 5.0,
  "Volatile_to_Fixed_Acidity": 0.4,
  "Alcohol_x_CitricAcid": 0.75,
  "ResidualSugar_x_Alcohol": 2.0,
  "pH_squared": 10.24
}

WHY JSON?
---------
✓ Easy for both humans and computers to read
✓ Works in all programming languages
✓ Lightweight (doesn't take much internet bandwidth)
✓ Industry standard for APIs

📚 PART 5: STEP-BY-STEP - HOW OUR SYSTEM WORKS
================================================================================

When you make a prediction, here's what happens:

STEP 1: User Opens Home Page
   👤 User: Visits http://localhost:5000/
   📱 Browser: Sends GET request
   🐍 Flask: Runs the home() function
   📄 Flask: Searches for templates/home.html
   🌐 Browser: Displays the beautiful home page

STEP 2: User Opens Postman
   💻 Postman: Opens on user's computer
   📝 User: Creates NEW request
   ⚙️ User: Sets method to POST
   🔗 User: Enters URL: http://localhost:5000/predict_api

STEP 3: User Sends JSON Data
   📋 User: Copies example JSON from home.html
   ✏️ User: Modifies values (e.g., Alcohol_log: 3.0)
   `🔘 User: Clicks "Send" button in Postman
   📤 Postman: Sends HTTP POST request with JSON data

STEP 4: Flask Receives Request
   🐍 Flask: @app.route('/predict_api', methods=['POST']) catches it
   🎯 Flask: predict_api() function runs
   📨 Flask: data = request.json (gets the JSON data)
   ✅ Flask: Validates that all required features are present

STEP 5: Model Makes Prediction
   🔢 Flask: Converts JSON to NumPy array in correct feature order
   🤖 Flask: trained_model.predict(features_array) runs
   🎉 Flask: Gets predicted price from model
   💰 Example: Predicted price = $45.73

STEP 6: Flask Sends Response Back
   📦 Flask: Creates response dictionary with prediction
   🔄 Flask: jsonify() converts dictionary to JSON
   📨 Flask: Sends response back to Postman
   HTTP Status: 200 (means success)

STEP 7: Postman Displays Result
   📊 Postman: Receives JSON response
   💻 Screen: Shows the prediction result
   🎯 Result: {
                 "success": true,
                 "predicted_price": 45.73,
                 "price_formatted": "$45.73"
               }

📚 PART 7: HOW TO USE POSTMAN (Detailed)
================================================================================

PREREQUISITE:
   Download Postman: https://www.postman.com/downloads/

STEP 1: Open Postman
   Click the Postman app to launch it

STEP 2: Create a New Request
   Click: "+ New" at the top left
   Select: "HTTP Request"
   
   OR
   
   Click: "Create" button on the left sidebar
   Select: "HTTP Request"

STEP 3: Set Request Method
   Look at the dropdown that says "GET" (top left of input area)
   Click it and select "POST"
   
   Why POST?
   → We're SENDING data TO the server
   → POST is for creating/submitting data
   → GET is for retrieving data

STEP 4: Enter the URL
   In the URL field (next to POST), type:
   http://localhost:5000/predict_api
   
   Note: Must have http:// (protocol)
   Note: localhost = your computer (127.0.0.1)
   Note: 5000 = the port Flask runs on

STEP 5: Configure Body
   Click the "Body" tab (below the URL field)
   Make sure "raw" is selected (radial button/checkbox on left)
   On the right side, change from "Text" to "JSON"
   
   Why?
   → raw = send text directly, don't encode it
   → JSON = treat our text as JSON format

STEP 6: Paste Example Data
   In the large text area, paste:
   
   {
     "Alcohol_log": 2.5,
     "Volatile_Acidity_log_boxcox": 0.8,
     "Citric_Acid": 0.3,
     "Chlorides": 0.05,
     "pH": 3.2,
     "Sulfites": 15,
     "Alcohol_to_Acidity": 5.0,
     "Volatile_to_Fixed_Acidity": 0.4,
     "Alcohol_x_CitricAcid": 0.75,
     "ResidualSugar_x_Alcohol": 2.0,
     "pH_squared": 10.24
   }

STEP 7: Send Request
   Click the blue "Send" button on the right
   Wait a moment for the response...

STEP 8: View Response
   Below your request, you'll see the response
   
   SUCCESS Response (Status: 200):
   {
     "success": true,
     "message": "✅ Prediction successful!",
     "predicted_price": 45.73,
     "price_formatted": "$45.73"
   }
   
   If it shows status 200 and the response has "success": true
   → CONGRATULATIONS! Your API works!

STEP 9: Modify and Test Again
   Change values in the JSON, e.g.:
   "Alcohol_log": 3.5  (instead of 2.5)
   Click Send again
   Get new prediction with new input

COMMON MISTAKES:
   ❌ Wrong URL (typo or wrong port)
      Fix: Copy URL exactly: http://localhost:5000/predict_api
   
   ❌ Using GET instead of POST
      Fix: Change dropdown to POST
   
   ❌ Invalid JSON (forgetting comma, wrong quotes)
      Fix: Postman will show error. Check syntax.
   
   ❌ Missing required features in JSON
      Fix: Include all 11 features with numeric values
   
   ❌ Feature names misspelled (case matters!)
      Fix: Use exact names: Alcohol_log NOT alcohol_log
   
   ❌ Flask not running
      Fix: Make sure terminal shows "Server running at..."

📚 PART 8: CODE EXPLANATION - app.py
================================================================================

IMPORTS:
--------
from flask import Flask, render_template, request, jsonify
│
├─ Flask: Main class to create web app
├─ render_template: Load HTML files from templates/
├─ request: Access incoming data from Postman
└─ jsonify: Convert Python dictionaries to JSON

import pickle
   → Load binary files (our saved ML model)

import numpy as np
   → Numerical operations, arrays

import pandas as pd
   → Data manipulation (though not used in this file)

CREATING THE APP:
-----------------
app = Flask(__name__)

__name__ = current module name
This tells Flask where to find templates/ and static/ folders

LOADING THE MODEL:
------------------
with open('WinePricePredictionP1/randomforest_model.pkl', 'rb') as model_file:
    trained_model = pickle.load(model_file)

'rb' = read binary (reading a binary/compiled file)
pickle.load() = deserialize (convert binary back to Python object)
Now trained_model = our trained RandomForest model

LOADING FEATURE NAMES:
----------------------
Why are feature names important?
  Model was trained with features in order: [F1, F2, F3, F4...]
  If we send [F2, F1, F3, F4...], predictions will be WRONG!
  
  Example:
    Trained on: [Alcohol, pH, Acidity]
    If we send: [pH, Alcohol, Acidity]
    Model thinks: pH=Alcohol value, Alcohol=pH value, Acidity=Acidity
    Result: WRONG PREDICTION!

Solution: Always use the SAME feature order

HOME ROUTE:
-----------
@app.route('/')
def home():
    return render_template('home.html')

@app.route('/') = URL is http://localhost:5000/
def home(): = function name (can be anything)
render_template('home.html') = find templates/home.html and send it

PREDICT API ROUTE:
------------------
@app.route('/predict_api', methods=['POST'])
def predict_api():

@app.route('/predict_api', = URL is http://localhost:5000/predict_api
methods=['POST'] = only accept POST requests
def predict_api(): = function name

RECEIVING DATA:
---------------
data = request.json

request.json = the JSON data Postman sent
Example: {"Alcohol_log": 2.5, "pH": 3.2, ...}

VALIDATING DATA:
----------------
if not data:
    return jsonify({"error": "No data received."}), 400

Checks if user sent empty JSON
400 = "Bad Request" HTTP status code (client error)

EXTRACTING FEATURES:
--------------------
for feature in feature_names:
    if feature not in data:
        return jsonify({"error": f"Missing feature: {feature}"}), 400
    features_list.append(data[feature])

Loop through each required feature in correct order
Check if user provided that feature
If missing, return error
If present, add its value to list

RESHAPING DATA:
---------------
features_array = np.array(features_list).reshape(1, -1)

np.array() = convert list to NumPy array
reshape(1, -1) = reshape to (1, num_features)
  1 = 1 sample (1 wine)
  -1 = auto-calculate number of columns

Why reshape?
  Model.predict() expects 2D array (rows, columns)
  Our list is 1D, so we reshape to 2D

MAKING PREDICTION:
------------------
predicted_price = trained_model.predict(features_array)[0]

trained_model.predict() = uses the model to predict
Returns array: [45.73]
[0] = extract first element: 45.73

RETURNING RESPONSE:
-------------------
response = {
    "success": True,
    "predicted_price": float(predicted_price),
    ...
}
return jsonify(response), 200

jsonify() = converts dictionary to JSON format
200 = HTTP status code "OK" (success)

ERROR HANDLING:
---------------
except Exception as error:
    return jsonify({"success": False, "error": str(error)}), 500

500 = "Internal Server Error" (something went wrong on our end)
If ANY error occurs, we catch it and return a friendly error message

RUNNING THE SERVER:
-------------------
if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=True)

if __name__ == '__main__' = only run this if script is run directly
host='0.0.0.0' = listen on all network interfaces
port=5000 = run on port 5000
debug=True = show errors, auto-restart on code changes

📚 PART 9: HTTP STATUS CODES
================================================================================

Status codes tell you whether the request succeeded or failed:

SUCCESS CODES (2xx):
  200 OK → Request succeeded, data returned
  201 Created → New resource created
  204 No Content → Success but no data to return

CLIENT ERROR CODES (4xx):
  400 Bad Request → Invalid data, malformed JSON
  401 Unauthorized → Need authentication
  404 Not Found → URL doesn't exist
  405 Method Not Allowed → Wrong HTTP method (GET vs POST)

SERVER ERROR CODES (5xx):
  500 Internal Server Error → Something crashed on server
  502 Bad Gateway → Server down or unreachable
  503 Service Unavailable → Server overloaded

IN OUR APP:
  200 = Prediction successful
  400 = Missing features or invalid JSON
  500 = Error in model prediction

📚 PART 10: TIPS & COMMON ISSUES
================================================================================

TIP 1: Feature Order Matters!
   The model is very strict about feature order.
   Always use the exact feature names in the provided example.
   One typo = wrong or crashed prediction.

TIP 2: Data Types Must Be Correct
   All values should be numbers (int or float)
   Don't send: "Alcohol_log": "hello" (string instead of number)
   Use: "Alcohol_log": 2.5 (number)

TIP 3: Keep Terminal Open When Testing
   Flask runs in the terminal
   If you close the terminal, Flask stops running
   Postman will fail to connect
   Solution: Keep terminal open in background

TIP 4: Multiple Predictions
   You can send as many predictions as you want
   Each one will use the same model
   Good for testing with different wine properties

TIP 5: Production vs Development
   Currently: debug=True (development mode)
   Shows error details, auto-restarts on code changes
   For production: debug=False (security)
   In production, use a proper server like Gunicorn

ISSUE 1: "Connection Refused"
   Error: Unable to connect to http://localhost:5000
   Cause: Flask not running
   Fix: Make sure you ran "python app.py"

ISSUE 2: "404 Not Found"
   Error: Postman shows "Cannot POST /predict_api"
   Cause: Wrong URL or Flask not started
   Fix: Check URL is exactly: http://localhost:5000/predict_api

ISSUE 3: "400 Bad Request - Missing feature"
   Error: {"error": "Missing feature: Alcohol_log"}
   Cause: Forgot to include a required feature in JSON
   Fix: Copy example JSON from home.html exactly

ISSUE 4: "Invalid JSON"
   Error: Postman shows JSON syntax error
   Cause: Missing comma, wrong quotes, etc.
   Fix: Use Postman's error message to find and fix syntax

ISSUE 5: Prediction makes no sense
   Error: Predicted price is -999 or 999999
   Cause: Values are way outside training range
   Fix: Use reasonable values similar to example
