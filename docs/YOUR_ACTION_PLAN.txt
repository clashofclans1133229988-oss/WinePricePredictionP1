SECTION 1: CLEAN UP YOUR GITHUB REPO (5 mins)
──────────────────────────────────────────────

Delete old Heroku deployment:

  1. In VS Code, open .github/workflows/deploy.yml
  2. Right-click file → Delete
  3. Confirm deletion
  4. In terminal:
     $ git add .
     $ git commit -m "Remove Heroku deployment"
     $ git push origin main

Done! Old Heroku workflow removed!

5. Click "Create a new project"

7. Select your wine-price-prediction repo
8. Click "Deploy"
9. WATCH THE LOGS (takes 1-2 minutes)

Done! App deployed to Railway!

────────────────────────────────────────────────

1. Go to https://fly.io
2. Click "Get Started"
3. Sign up (YES need credit card, won't charge free tier)
4. Verify email
5. Install fly CLI:
   $ curl https://fly.io/install.sh | sh
6. Login:
   $ fly auth login
7. In your project directory:
   $ fly launch
   └─ Choose app name: wine-price-app
   └─ Region: us-west (or nearby)
   └─ Postgres database? NO
   └─ Deploy now? YES
8. Wait for deployment (watch logs)

Done! App deployed!

================================================================================
🆘 QUICK TROUBLESHOOTING
================================================================================

PROBLEM: GitHub Actions CI fails
SOLUTION: Make sure code runs locally first
  $ python -m py_compile app.py

PROBLEM: Docker build fails on GitHub
SOLUTION: Test locally
  $ docker build -t test .

PROBLEM: Deployment works but app shows error
SOLUTION: Check platform logs
  Railway: Dashboard → project → logs
  Fly: $ fly logs

PROBLEM: Boss can't access link
SOLUTION:
  1. Make sure you sent correct URL
  2. App might be sleeping (Railway free tier)
  3. Test URL works for you first
  4. Refresh page

PROBLEM: Changes don't auto-deploy
SOLUTION:
  1. Make sure push was successful
  2. GitHub Actions workflow ran?
  3. Check GitHub Actions tab
  4. Check platform dashboard
