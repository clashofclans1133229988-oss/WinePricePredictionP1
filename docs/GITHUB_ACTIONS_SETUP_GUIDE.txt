📚 PART 2: CREATE GITHUB SECRETS
================================================================================

GitHub Secrets are encrypted environment variables. GitHub Actions workflows
access them, but they're hidden from logs.

HOW TO CREATE SECRETS:
──────────────────────

1. Go to your GitHub repository
2. Click "Settings" tab (top navigation)
3. Left sidebar: Click "Secrets and variables" → "Actions"
4. Click "New repository secret" button
5. Enter secret name and value
6. Click "Add secret"
7. Repeat for all secrets

REQUIRED SECRETS FOR YOUR PROJECT:
──────────────────────────────────

SECRET 1: DOCKER_USERNAME
──────────────────────────
What it is:
  └─ Your Docker Hub username

Where to get it:
  1. Log in to https://hub.docker.com
  2. Click your profile icon (top right)
  3. Click "Account Settings"
  4. Your username is displayed

How to add:
  1. GitHub Settings → Secrets and variables → Actions
  2. "New repository secret"
  3. Name: DOCKER_USERNAME
  4. Value: your-docker-username (exactly as on Docker Hub)
  5. Click "Add secret"

Example:
  If your Docker Hub username is "john.doe"
  Then value = "john.doe"

SECRET 2: DOCKER_PASSWORD
──────────────────────────
What it is:
  └─ Docker Hub authentication token (safer than password)

⚠️ IMPORTANT: Use Access Token, NOT your Docker Hub password!
  └─ Passwords are less secure, tokens are revocable

How to create Docker Hub Access Token:
  1. Log in to https://hub.docker.com
  2. Click your profile icon → "Account Settings"
  3. Left sidebar: Click "Security" or "Personal access tokens"
  4. Click "Create access token" or "Generate new token"
  5. Name it: "github-actions" or similar
  6. Permissions: Read & Write (or just Read if available)
  7. Copy the token
  8. Save it somewhere safe!
  9. ⚠️ This token won't show again!

How to add to GitHub:
  1. GitHub Settings → Secrets and variables → Actions
  2. "New repository secret"
  3. Name: DOCKER_PASSWORD
  4. Value: (paste your Docker Hub access token)
  5. Click "Add secret"

Example token format:
  dckr_pat_xxxxxxxxxxxxxxxxxxxxxxxxxxxxx

SECRET 3: HEROKU_API_KEY
────────────────────────
What it is:
  └─ Your Heroku authentication key

How to create:
  1. Log in to https://dashboard.heroku.com
  2. Click your profile icon (top right)
  3. Click "Account settings"
  4. Scroll down to "API Key" section
  5. Click "Reveal" or "Generate new..."
  6. Copy the API key
  7. Save it somewhere safe!

How to add to GitHub:
  1. GitHub Settings → Secrets and variables → Actions
  2. "New repository secret"
  3. Name: HEROKU_API_KEY
  4. Value: (paste your Heroku API key)
  5. Click "Add secret"

Example format:
  a1b2c3d4-e5f6-g7h8-i9j0-k1l2m3n4o5p6

SECRET 4: HEROKU_EMAIL
───────────────────────
What it is:
  └─ Your Heroku account email

How to add:
  1. GitHub Settings → Secrets and variables → Actions
  2. "New repository secret"
  3. Name: HEROKU_EMAIL
  4. Value: your-email@example.com
  5. Click "Add secret"

Example:
  john@example.com

SECRET 5: HEROKU_APP_NAME
──────────────────────────
What it is:
  └─ The name of your Heroku app

How to create Heroku app:
  1. Go to https://dashboard.heroku.com
  2. Click "Create new app"
  3. App name: wine-price-app (or your choice)
  4. Region: United States or Europe
  5. Click "Create app"
  6. Copy the app name

How to add to GitHub:
  1. GitHub Settings → Secrets and variables → Actions
  2. "New repository secret"
  3. Name: HEROKU_APP_NAME
  4. Value: your-app-name (e.g., wine-price-app)
  5. Click "Add secret"

Example:
  wine-price-app

OPTIONAL SECRET: SLACK_WEBHOOK
────────────────────────────────
What it is:
  └─ Webhook URL for Slack notifications (optional)

Skip this if you don't use Slack!

How to create:
  1. Create Slack app on https://api.slack.com/apps
  2. Create Incoming Webhook
  3. Post webhook URL to channel
  4. Copy webhook URL

How to add:
  1. GitHub Settings → Secrets and variables → Actions
  2. "New repository secret"
  3. Name: SLACK_WEBHOOK
  4. Value: https://hooks.slack.com/services/...
  5. Click "Add secret"

VERIFY ALL SECRETS ADDED:
─────────────────────────

Go to GitHub Settings → Secrets and variables → Actions

You should see these secrets:
  ✓ DOCKER_USERNAME
  ✓ DOCKER_PASSWORD
  ✓ HEROKU_API_KEY
  ✓ HEROKU_EMAIL
  ✓ HEROKU_APP_NAME
  ✓ (Optional) SLACK_WEBHOOK

📚 PART 3: ENABLE GITHUB ACTIONS
================================================================================

GitHub Actions should be enabled by default, but verify:

1. Go to your repository
2. Click "Actions" tab (top navigation)
3. You should see "Workflows" section

If you see message "Actions is disabled":
  1. Go to Settings tab
  2. Left sidebar: Actions → General
  3. Choose: "Allow all actions and reusable workflows"
  4. Click "Save"

📚 PART 4: CONFIGURE BRANCH PROTECTION
================================================================================

This ensures bad code doesn't merge to main branch.

Setup:
  1. Go to Settings → Branches
  2. Under "Branch protection rules" click "Add rule"
  3. Pattern: main
  4. Check: "Require a pull request before merging"
  5. Check: "Require status checks to pass before merging"
  6. Check: "CI Testing" workflow
  7. Check: (optional) "Build and Push Docker Image"
  8. Click "Create"

Result:
  ├─ Can't push directly to main
  ├─ Must create pull request
  ├─ Workflows must pass
  ├─ Code review required
  └─ Main branch stays clean!

📚 PART 6: UNDERSTANDING WORKFLOW STATUS
================================================================================

What different status indicators mean:

ON PULL REQUEST:
───────────────

✅ All checks passed
  └─ Safe to merge
  └─ Click "Merge pull request"

❌ Some checks failed
  └─ Fix errors first
  └─ Push fixes
  └─ Workflow re-runs automatically
  └─ When all pass, then merge

⏳ Checks in progress
  └─ Still running, wait...
  └─ Takes 1-5 minutes
  └─ Don't merge yet

⊘ Checks skipped
  └─ Workflow didn't run
  └─ Check workflow settings
  └─ Might be disabled

DETAILED STATUS:
────────────────

Click workflow name to see:
  ├─ Which steps are running/passed/failed
  ├─ Logs for each step
  ├─ Errors with full details
  ├─ Duration of each step
  └─ Environment variables used

ERROR HANDLING:
───────────────

If workflow fails:
  1. Click failed workflow
  2. Scroll to failed step
  3. Read error message
  4. Fix issue
  5. Commit and push
  6. Workflow runs again

Common issues:
  ├─ Missing pip packages → Add to requirements.txt
  ├─ Code syntax errors → Fix Python code
  ├─ Missing secrets → Add to GitHub Secrets
  ├─ Wrong secret value → Verify in Secrets
  └─ Docker login failed → Check Docker Hub token

📚 PART 8: WORKFLOW SCHEDULE (OPTIONAL)
================================================================================

You can schedule workflows to run automatically (like cron jobs).

Example: Nightly tests

Edit .github/workflows/ci.yml:

  on:
    push:
      branches: [main, develop]
    pull_request:
      branches: [main]
    schedule:
      # Run at 2 AM UTC every day
      - cron: '0 2 * * *'

Cron syntax:
  minute hour day month day-of-week

More examples:
  ├─ '0 2 * * *' = 2 AM every day
  ├─ '0 */6 * * *' = Every 6 hours
  ├─ '0 0 * * 1' = Every Monday at midnight
  ├─ '30 1 * * *' = 1:30 AM every day
  └─ '0 0 1 * *' = 1st of month at midnight

📚 PART 10: COMMON SETUP ISSUES & SOLUTIONS
================================================================================

ISSUE 1: "Secrets not found" error
───────────────────────────────────

Error message:
  Docker username is empty

Fix:
  1. Go to Settings → Secrets and variables
  2. Verify all 5 secrets are added
  3. Check secret names exactly match:
     ├─ DOCKER_USERNAME
     ├─ DOCKER_PASSWORD
     ├─ HEROKU_API_KEY
     ├─ HEROKU_EMAIL
     └─ HEROKU_APP_NAME
  4. Re-add any missing secrets

ISSUE 2: Workflow doesn't run
──────────────────────────────

Causes:
  ├─ Workflow file has YAML syntax error
  ├─ File path wrong (not .github/workflows/)
  ├─ GitHub Actions disabled
  ├─ Branch protection blocking it

Fix:
  1. Check workflow file YAML syntax (use online validator)
  2. Verify file in .github/workflows/ directory
  3. Go to Settings → Actions → Ensure enabled
  4. Check branch protection rules

ISSUE 3: "Authentication failed" in Docker step
──────────────────────────────────────────────

Cause:
  ├─ Wrong Docker Hub credentials
  ├─ Token expired
  ├─ Typo in username/password

Fix:
  1. Verify DOCKER_USERNAME and DOCKER_PASSWORD
  2. Recreate Docker Hub access token
  3. Update GitHub Secrets with new token
  4. Re-run workflow

ISSUE 4: "Heroku app not found" error
──────────────────────────────────────

Cause:
  ├─ Wrong HEROKU_APP_NAME
  ├─ App deleted
  ├─ Heroku API key invalid

Fix:
  1. Verify Heroku app exists
  2. Go to https://dashboard.heroku.com
  3. Copy exact app name
  4. Update HEROKU_APP_NAME secret
  5. Verify HEROKU_API_KEY is current

ISSUE 5: Pull request won't merge despite passing checks
──────────────────────────────────────────────────────────

Cause:
  ├─ Missing required approvals
  ├─ Branch protection rule
  ├─ Admin restriction

Fix:
  1. Ask code reviewer to approve
  2. Check Settings → Branches → Protection rules
  3. Ensure all required conditions met
