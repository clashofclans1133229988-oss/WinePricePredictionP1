📚 PART 2: HOW GITHUB FITS IN
================================================================================

GITHUB'S ROLE IN DEPLOYMENT:
────────────────────────────

GitHub DOESN'T host your app, BUT:

┌─────────────────────────────────────────┐
│ You push code to GitHub                │
│              ↓                          │
│ GitHub Actions workflow triggers       │
│              ↓                          │
│ 1. Build Docker image                  │
│ 2. Push to Docker registry              │
│ 3. Tell hosting platform to deploy      │
│              ↓                          │
│ Hosting platform (Railway/Fly.io)      │
│ pulls Docker image                      │
│              ↓                          │
│ App runs and serves users               │
│              ↓                          │
│ Share link: https://my-app.railway.app │
└─────────────────────────────────────────┘

SO:
  ├─ GitHub = stores code & triggers deployment
  ├─ Railway/Fly.io = actually runs your app
  ├─ Docker = packages your app for deployment
  └─ All work together!

CAN GITHUB DIRECTLY HOST?
──────────────────────────

No, but:
  ✅ GitHub Actions can BUILD
  ✅ GitHub Actions can PUSH to registry
  ✅ GitHub Actions can TRIGGER deployment
  ✅ Hosting platform RUNS the app

Think of it:
  ├─ GitHub = Kitchen (where you prepare)
  ├─ Docker = Delivery box (what you ship in)
  ├─ Railway = Restaurant (where it's served)
  └─ Users = Customers (eat your food)

If you want more control/better performance:

STEP 2: INSTALL FLY CLI
════════════════════════

Mac/Linux:
  $ curl https://fly.io/install.sh | sh

Windows:
  $ iwr https://fly.io/install.ps1 -useb | iex

Verify: fly version

STEP 3: LOGIN TO FLY
────────────────────

$ fly auth login

Opens browser to login!

STEP 4: LAUNCH APP
──────────────────

In your project directory:

$ fly launch

Follow prompts:
  ├─ App name: wine-price-app
  ├─ Organization: choose personal
  ├─ Region: closest to you (or us-west for USA)
  ├─ Would you like to set up a Postgres database? NO
  └─ Deploy now? YES

STEP 5: DEPLOY!
───────────────

$ fly deploy

Watches logs as it:
  1. Builds Docker image
  2. Uploads to Fly
  3. Starts container
  4. Runs app

When done, see: https://wine-price-app.fly.dev

STEP 6: VIEW APP
────────────────

$ fly open

Opens your app in browser!

STEP 7: SHARE LINK
──────────────────

Share: https://wine-price-app.fly.dev with your boss!

AUTO-DEPLOY WITH GITHUB ACTIONS:
─────────────────────────────────

Add to .github/workflows/deploy-fly.yml:

```yaml
name: Deploy to Fly.io

on:
  push:
    branches: [main]

env:
  FLY_API_TOKEN: ${{ secrets.FLY_API_TOKEN }}

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Deploy to Fly.io
        uses: superfly/flyctl-actions@master
        with:
          args: "deploy"
```

Then:
  1. Create fly token: fly tokens create deploy
  2. Add to GitHub secrets: FLY_API_TOKEN
  3. Every push auto-deploys!
