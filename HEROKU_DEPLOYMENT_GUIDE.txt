📚 PART 1: WHAT IS A PROCFILE?
================================================================================

SIMPLE DEFINITION:
──────────────────
A Procfile tells Heroku what commands to run when your app starts.

Think of it as INSTRUCTIONS for Heroku:
  "Hey Heroku, here's how to run my app!"

FILENAME: Procfile (no extension, no .txt)
LOCATION: Root folder of your project (same level as app.py)

WHAT'S IN IT:
─────────────
web: gunicorn app:app

Breaking it down:
  ├─ "web" = process type (handle HTTP requests)
  ├─ "gunicorn" = production web server
  ├─ "app:app" = module:variable
  │   ├─ First "app" = your app.py file
  │   └─ Second "app" = the Flask application object (app = Flask(__name__))
  └─ Together: "run the Flask app from app.py using Gunicorn"

================================================================================
📚 PART 2: WHY GUNICORN INSTEAD OF FLASK?
================================================================================

LOCAL DEVELOPMENT (your computer):
──────────────────────────────────
You use: python app.py
  └─ Runs Flask development server
  └─ Good for testing
  └─ Single process (slow for multiple users)
  └─ Debug mode on (shows errors)
  └─ NOT suitable for production


PRODUCTION (Heroku hosting):
───────────────────────────
You use: gunicorn app:app
  └─ Runs Gunicorn production server
  └─ Handles multiple requests simultaneously
  └─ More stable and secure
  └─ Production-grade performance
  └─ What professional apps use

ANALOGY:
────────
Flask dev server = Test kitchen (one chef, slow)
Gunicorn = Professional restaurant (many chefs, fast, efficient)

📚 PART 8: WHAT IF DEPLOYMENT FAILS?
================================================================================

ERROR 1: "fatal: 'heroku' does not appear to be a git remote"
────────────────────────────────────────────────────────────

Cause:
  You didn't run "heroku create" first

Solution:
  1. Command: heroku create your-app-name
  2. Command: git push heroku main


ERROR 2: "Permission denied (publickey)"
────────────────────────────────────────

Cause:
  SSH key not configured

Solution:
  1. Command: heroku login
  2. Command: heroku keys:add
  3. Try again: git push heroku main


ERROR 3: Application Crash
──────────────────────────

See logs:
  Command: heroku logs --tail

Common causes & fixes:
  ├─ Missing package: Add to requirements.txt
  ├─ Python version wrong: Check runtime.txt
  ├─ Port not configured: Heroku sets port automatically
  ├─ File path wrong: Pickle files must be in right folder
  └─ App name wrong in Procfile: Check "app:app" matches


ERROR 4: "No such file or directory: randomforest_model.pkl"
───────────────────────────────────────────────────────────

Cause:
  Heroku can't find pickle files

Solution:
  Check file paths in app.py:
    ```python
    with open('WinePricePredictionP1/randomforest_model.pkl', 'rb') as f:
        model = pickle.load(f)
    ```
  
  Make sure:
    ├─ Path is relative (not absolute)
    ├─ Folder name matches exactly
    ├─ File name matches exactly (case sensitive!)
    └─ .gitignore doesn't exclude pickle files


ERROR 5: "ModuleNotFoundError: No module named 'flask'"
──────────────────────────────────────────────────────

Cause:
  requirements.txt not uploaded or not read

Solution:
  1. Verify requirements.txt in project root
  2. Run: git add requirements.txt
  3. Run: git commit -m "Update requirements"
  4. Run: git push heroku main


VIEW LOGS FOR DEBUGGING:
→ Command: heroku logs --tail
  Shows real-time logs
  Press Ctrl+C to exit

→ Command: heroku logs --num=50
  Shows last 50 lines

📚 PART 9: UNDERSTANDING PROCFILE SYNTAX
================================================================================

Basic Format:
──────────────

<process-type>: <command>

Examples:

1. Flask Development (NOT for production)
   ──────────────────────────────────────
   web: python app.py
   └─ Uses Flask development server
   └─ Only one worker
   └─ Slow

2. Flask with Gunicorn (RECOMMENDED)
   ────────────────────────────────────
   web: gunicorn app:app
   └─ Uses production Gunicorn server
   └─ Multiple workers
   └─ Fast & stable ✅

3. With Custom Port
   ──────────────────
   web: gunicorn --bind 0.0.0.0:$PORT app:app
   └─ Heroku assigns $PORT automatically
   └─ Usually not needed (gunicorn handles it)

4. Multiple Processes
   ─────────────────
   web: gunicorn app:app
   worker: python worker.py
   └─ web process: handles HTTP requests
   └─ worker process: background jobs

WHAT "app:app" MEANS:
────────────────────

Looking at your app.py:

```python
from flask import Flask

app = Flask(__name__)  ← This is the Flask object

@app.route('/')
def home():
    return render_template('home.html')

if __name__ == '__main__':
    app.run(...)
```

In Procfile: gunicorn app:app
  └─ First "app" = the module/file (app.py)
  └─ Second "app" = the Flask object (app = Flask(__name__))
  └─ Together: "run the Flask object from app.py using Gunicorn"
